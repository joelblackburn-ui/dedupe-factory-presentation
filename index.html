Where Duplicates Come From: Update Gaps

Missing / Non-Standardised Data

Unable to find existing record → create new

  • Domain missing or incorrect
  • Name variations (Ltd vs Limited)
  • No LinkedIn URL for matching
  • Different regional formats

Corporate Changes

Looks new but isn't

  • Company rebrands (Camelot → Allwyn)
  • Acquisitions & mergers
  • Domain changes
  • Subsidiary relationships

Result: Same company exists multiple times → fragmented history, inaccurate reporting, wasted sales effort

How Matching Works: Foundation

Code-Based Matching

Run in containers, fast & deterministic:

Exact domain match High signal
LinkedIn URL match High signal
Fuzzy name similarity Medium signal
Address / phone match Supporting signal

Search Strategy

For each company, run targeted searches:

  • Search by domain → find matches
  • Search by company name (fuzzy) → find variations
  • Search by LinkedIn URL → find matches
  • Search by phone → find matches
  • Check domain redirects → detect rebrands

How Matching Works: Agentic Scoring

Beyond basic matching — agents conduct research and validation:

Domain Analysis

  • Check if domains redirect to each other
  • Verify website content matches
  • Detect rebrand signals

M&A Research

  • Search for merger/acquisition news
  • Confirm parent-child relationships
  • Identify corporate structure

Adaptive Enrichment

  • Request additional data when uncertain
  • Scrape websites for verification
  • Cross-reference multiple sources

Validation Loop

The Validator agent reviews the Scorer's assessment. If confidence is borderline or evidence is conflicting, it can trigger re-enrichment or escalate to human review with specific questions.

Security & Safety

Protected Records

NEVER auto-merged:

  • Customer records
  • In-pipeline deals
  • Partner accounts

Protected records always require human review

Rollback Capability

  • Pre-merge snapshots of all records
  • All associations captured
  • Full restore within minutes
  • Complete audit trail

Infrastructure

  • Cloudflare Workers for rate limiting
  • Separate read/write API access
  • Anomaly detection
  • Emergency stop triggers

Test Environment First

Before production rollout, we create a test environment that mirrors HubSpot functionality with representative endpoints and data. All agents are validated against this sandbox before touching live data.
